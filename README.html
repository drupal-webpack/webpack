Use webpack to bundle javascript files, transpile them and include modules from a site-wide package.json file.

Should I use it now?

No, unless you want to test it out and contribute. Right now it's just a working prototype without any settings. The values are hardcoded for a composer project with a package.json in the repository root (with at least webpack and webpack-cli) and libraries in modules/custom.

What it does?

This module lets developers write their Drupal libraries in modern javascript and import modules from npm.

How it works?

Right now it decorates the asset.js.collection_renderer service and bundles all the js libraries matching criteria into a single file. This is going to change, as decorating the asset.resolver service seems to be much more promising.

How can I contribute?

Suggest features or improvements by creating a new issue. Alternatively,
  • Find a TODO comment in code.
  • Search for an existing task for this TODO item in the issue queue.
    • If there is one, check the progress and help there if possible.
    • Otherwise, create a new issue with the name of the todo and assign it to yourself.
Usually, it's a good idea to discuss the problem in the issue before writing patches. Ping @blazey in #webpack on drupalchat.eu (or slack).
